static : 정적 메모리 영역
- 프로그램 실행 시 static이 붙은 변수/메서드를
->static 메모리 영역에 할당(클래스.변수 / 클래스명. 메서드명 형태로 할당)
->객체(인스턴스) 생성 없이 클래스명.필드명 / 클래스명.메서드명() 으로 직접 접근이 가능하다

-> 프로그램 내에서 공유 

static() -> non-static 호출 X
static() -> static 호출 O
non-static ->non-static호출 O
non-static -> static호출 O

메서드를 호출하려면 클래스 객체생성

sysout -함수호출안해도 바로실행
스캐너,메인()내부,static{}블록내부,클래스필드에서 메서드호출

[내부 클래스]

1. 인스턴스 내부 클래스
- 외부 클래스의 멤버 처럼 활용되는 내부 클래스
- 외부 클래스의 인스턴스(닭)가 존재 해야지만 
  내부 클래스 인스턴스(달걀) 생성 가능 

2.정적(static) 내부 클래스
- 외부 인스턴스 생성 없이
  내부 인스턴스 생성 가능!
  (new 외부클래스명.내부클래스명())

3. 지역 내부 클래스
- 외부 클래스의 멤버 메서드 내에 존재하는 내부 클래스
- 외부 메서드의 내부에서만 생성, 사용 가능

4. 익명 내부 클래스
- 이름이 없는 내부 클래스
- 인터페이스, 추상클래스를 상속 받은 구현체를
  따로 만들지 않고 필요한 곳에서 클래스를 정의(추상 메서드 오버라이딩)
  형식의 클래스
- 일회용으로 주로 사용
